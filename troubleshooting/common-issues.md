# 常见问题排查

## 网速慢

### 排查步骤

1. **切换节点** — 晚高峰（晚上8-11点）建议用香港/日本节点，美国节点延迟高速度慢
2. **检查本地网络** — 关闭其他下载任务，测一下本地带宽
3. **更换协议** — 如果用 VMess，试试换 VLESS+Reality
4. **清除 DNS 缓存** — Windows: `ipconfig /flushdns`，Mac: `sudo dscacheutil -flushcache`
5. **测速** — 用 https://fast.com 或 https://speedtest.net 对比

### 常见原因
- 节点高峰期限速
- 节点距离太远
- 墙对特定线路做了 QoS 限速
- 本地运营商对代理做了限制

## 连接失败

### 排查步骤

1. **检查节点信息** — UUID、地址、端口是否输入正确
2. **检查端口** — 确认服务端口（如443）已放行
3. **检查防火墙** — VPS 和本地防火墙都要检查
4. **ping 测试** — `ping 你的VPS_IP` 看是否能通
5. **更换节点** — 当前节点可能被封

### Windows 特定
- 以管理员身份运行客户端
- 关闭 Windows Defender 防火墙测试
- 检查代理软件是否被拦截

---

## 被封了怎么办

### 情况一：订阅链接打不开
1. 联系机场客服获取新的订阅地址
2. 或在机场官网手动获取节点信息，手动导入

### 情况二：IP 被墙
- **机场节点**：换用其他节点
- **自建节点**：
  - 方案A：换一个 IP（换机房/换服务商）
  - 方案B：使用域名连接（域名前置）
  - 方案C：配置 CDN 隐藏真实 IP

### 情况三：端口被封
- 换一个端口（常用：443、80、8080）
- Reality 建议用 443 端口

### 情况四：账号被封
- 只能换账号，没有其他办法

---

## 防封建议

1. **不要用免费节点** — 免费节点易被封，且不安全
2. **不要长时间用同一节点** — 定期更换
3. **用 Reality 等伪装协议** — 大幅降低被封概率
4. **不要在公开场合分享节点** — 容易被滥用导致被封
5. **备用方案** — 保持2-3个可用节点，不把鸡蛋放一个篮子
